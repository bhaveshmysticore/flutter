Flutter / Dart Functions & Methods Guide

1. Basic Function Syntax

Basic structure of a function in Flutter (Dart):

returnType functionName(parameters) {
  // code
  return value;
}

Example:

int addNumbers(int a, int b) {
  return a + b;
}

2. Simple Function

void showMessage() {
  print("Hello Flutter");
}

Call the function:

showMessage();

3. Function With Parameters

void greetUser(String name) {
  print("Hello $name");
}

greetUser("Bhavesh");

4. Function With Return Value

double calculatePrice(double price, double tax) {
  return price + tax;
}

var total = calculatePrice(100, 18);

5. Arrow Function (Short Syntax)

Used for single-line logic.

int square(int number) => number * number;

6. Named Parameters (Common in Flutter)

void createUser({String? name, int? age}) {
  print("Name: $name");
  print("Age: $age");
}

createUser(name: "Bhavesh", age: 25);

7. Required Named Parameters

void createUser({
  required String name,
  required int age,
}) {
  print(name);
}

createUser(name: "Bhavesh", age: 25);

8. Default Parameter Values

void printMessage({String message = "Hello"}) {
  print(message);
}

printMessage();
printMessage(message: "Flutter");

9. Async Functions (Important in Flutter)

Used for API calls, Firebase, database operations.

Future fetchData() async {
  await Future.delayed(Duration(seconds: 2));
  return "Data Loaded";
}

10. Private Functions

Use underscore (_) to make a function private to the file.

void _loadUserData() {
  print("Loading user data");
}

11. Functions Inside Widgets

class HomeScreen extends StatelessWidget {

  void _showMessage() {
    print("Button clicked");
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: Center(
        child: ElevatedButton(
          onPressed: _showMessage,
          child: Text("Click"),
        ),
      ),
    );
  }
}

12. Callback Functions

ElevatedButton(
  onPressed: () {
    print("Pressed");
  },
)

Passing function reference:

ElevatedButton(
  onPressed: submitForm,
)

void submitForm() {
  print("Submitted");
}

13. Functions Returning Widgets

Widget buildButton() {
  return ElevatedButton(
    onPressed: () {},
    child: Text("Submit"),
  );
}

14. Generic Functions

T getFirstItem(List items) {
  return items.first;
}

15. Best Practices

  • Keep functions small and readable
  • Use meaningful function names
  • Avoid business logic inside UI widgets
  • Use async/await properly
  • Separate API logic into services

16. Real Flutter Example

class LoginController {

  Future loginUser({
    required String email,
    required String password,
  }) async {

    if (email.isEmpty || password.isEmpty) {
      return false;
    }

    var response = await ApiService.login(email, password);

    if (response.statusCode == 200) {
      return true;
    }

    return false;
  }
}

Conclusion

Writing good functions in Flutter improves readability, maintainability, and scalability of applications.

  • Use small functions
  • Use required named parameters
  • Use async/await for API calls
  • Separate logic from UI
  • Follow clean architecture

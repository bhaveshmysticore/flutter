Flutter Controllers & API Handling Guide

1. What is a Controller in Flutter?

A controller manages the business logic of the application. Instead of writing logic inside UI widgets, developers move logic to controllers. This keeps UI clean and improves maintainability.

  • Handle API calls
  • Manage state
  • Process business logic
  • Send data to UI

3. User Model

class UserModel {
  final int id;
  final String name;
  final String email;

  UserModel({
    required this.id,
    required this.name,
    required this.email,
  });

  factory UserModel.fromJson(Map json) {
    return UserModel(
      id: json['id'],
      name: json['name'],
      email: json['email'],
    );
  }
}

4. API Service (Common HTTP Logic)

import 'package:http/http.dart' as http;
import 'dart:convert';

class ApiService {

  static Future getRequest(String url) async {

    final response = await http.get(Uri.parse(url));

    if (response.statusCode == 200) {
      return jsonDecode(response.body);
    } else {
      throw Exception("API Failed");
    }

  }

}

5. API Layer (Feature Specific API)

class UserApi {

  static Future> fetchUsers() async {

    final data = await ApiService.getRequest(
      "https://jsonplaceholder.typicode.com/users"
    );

    List users = data;

    return users.map((e) => UserModel.fromJson(e)).toList();

  }

}

6. Controller Example

class UserController {

  List users = [];

  bool isLoading = false;

  Future loadUsers() async {

    try {

      isLoading = true;

      users = await UserApi.fetchUsers();

    } catch (e) {

      print(e);

    } finally {

      isLoading = false;

    }

  }

}

7. Using Controller in UI

class UserScreen extends StatefulWidget {
  @override
  _UserScreenState createState() => _UserScreenState();
}

class _UserScreenState extends State {

  final controller = UserController();

  @override
  void initState() {
    super.initState();
    controller.loadUsers();
  }

  @override
  Widget build(BuildContext context) {

    return Scaffold(
      appBar: AppBar(title: Text("Users")),

      body: ListView.builder(
        itemCount: controller.users.length,
        itemBuilder: (context,index){

          final user = controller.users[index];

          return ListTile(
            title: Text(user.name),
            subtitle: Text(user.email),
          );

        },
      ),
    );
  }
}

8. POST API Example

static Future createUser(String name,String email) async {

  final response = await http.post(
    Uri.parse("https://example.com/users"),
    body:{

      "email":email

  if(response.statusCode == 200){

9. Error Handling Best Practice

Future loadUsers() async {

  try {

    isLoading = true;

    users = await UserApi.fetchUsers();

  } catch (e) {

    print("Error: $e");

  } finally {

    isLoading = false;

  }

}

10. Best Practices Used in Production Apps

  • Separate API logic from UI
  • Use controllers or providers for business logic
  • Use models for API data
  • Use services for common network code
  • Handle API errors properly
  • Keep widgets clean and simple

Conclusion

Using controllers and separate API layers makes Flutter apps scalable and maintainable. This structure is used in most professional Flutter applications.
